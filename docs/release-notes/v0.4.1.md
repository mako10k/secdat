# secdat v0.4.1 Release Notes

## Highlights

- Adds `secdat help inject` as the direct CLI help topic for `exec --inject`
  rule grammar.
- Routes `--inject` parser errors directly to `help inject` instead of only
  pointing users back through `help exec`.
- Improves Japanese help text for the inject rule topic and refreshes the
  translation catalog.

## CLI and documentation

- Global help and `help exec` now advertise `help inject`.
- `help inject` explains layers, rule kinds, selector syntax, route picks,
  `secret:rename`, YAML policy files, `--bulk-gate`, and dry-run preflight
  examples.
- Topic help no longer advertises command-style `inject --help`; use
  `secdat help inject` or `secdat --help inject`.
- The command reference and README document the new inject help path.

## Validation

- The release was validated with the full `make check` regression suite.

## Versions

- The C project version is `0.4.1`.
- Python, Rust, and Node binding package versions track `0.4.1`.
- The Rust binding requires `libsecdat >= 0.4.1` through `pkg-config`.
