# secdat v0.3.0 Release Notes

## Highlights

- Adds explicit domain root migration and recovery for moved, deleted, or
  recreated directory roots without changing the domain ID or stored secrets.
- Adds root identity checks for newly created domains so normal `--dir` and cwd
  resolution fail closed when a registered root is replaced or becomes stale.
- Adds optional `secdat-fuse` mount support for exposing and updating selected
  secrets through a FUSE view.
- Hardens orphaned-domain reporting so JSON and SDK domain metadata cannot make
  stale registered roots appear unlocked through inherited sessions.

## C SDK and CLI

- The C project version is `0.3.0`.
- `domain move --from OLD_ROOT [--to NEW_ROOT]` can recover an existing domain
  at a new or recreated root while preserving the domain ID, stores, and
  secrets.
- `domain ls -l` and `domain ls --json` distinguish orphaned registered roots
  from ordinary locked or unlocked domains.
- `unlock --descendants` now fails closed with recovery guidance when an
  orphaned registered descendant is present in the subtree.
- `status --json` includes `resolution_error` so scripts can distinguish
  domain-resolution failure from an ordinary locked state.
- `exec` supports required-key guards and audit/preflight summaries for
  environment injection workflows.
- Relation and metadata commands gained searchable filters and refresh-target
  suggestions for leaked-key workflows.

## Bindings

- Python, Rust, and Node binding package versions now track `0.3.0`.
- The Rust binding requires `libsecdat >= 0.3.0` through `pkg-config`.
- SDK metadata list APIs expose key, store, and domain metadata without reading
  plaintext secret values.

## Known Constraints

- Root identity metadata is based on filesystem device/inode information. It is
  a stale-root safety check, not a cryptographic directory identity.
- `domain move` intentionally preserves the domain ID, so any still-active
  session scoped to that domain remains effective after explicit recovery.
- `secdat-fuse` is optional and depends on a build configured with FUSE3
  support.
