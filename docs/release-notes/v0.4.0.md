# secdat v0.4.0 Release Notes

## Highlights

- Introduces the `exec` environment injection engine with `--inject`,
  `--inject-file`, and pentad-based child environment planning.
- Adopts bulk-selection vocabulary (`bulk_select`, `--bulk-select`,
  `--bulk-gate`) and separates it from exec-only inject supply rules.
- Rejects legacy inject/bulk CLI names with actionable hints instead of silent
  aliases.
- Extends bash completion for `exec` to complete child commands and arguments
  after secdat options.

## Breaking changes

### Store attributes and policy fields

| Legacy | Canonical |
| --- | --- |
| `sandbox_inject` / `inject_bulk` | `bulk_select` |
| `sandbox_inject_entry` / `inject_bulk_entry` | `bulk_select_entry` |
| `sandbox_inject_value` / `inject_bulk_value` | `bulk_select_value` |

Token values remain `exclude`, `named`, and `include`. On-disk readers still
accept older field names during transition; writers emit canonical names only.

### CLI flags

| Legacy | Canonical | Notes |
| --- | --- | --- |
| `--sandbox-inject` / `--inject-bulk` | `--bulk-select` | `attr`, `set` |
| `--sandbox-injectable` / `--inject-bulk-gate` | `--bulk-gate` | `ls`, `list`, `export`, `fuse`, `exec` |
| `--inject-gate=sandbox` | `--bulk-gate` | `exec` |
| `exec --pattern` / `--pattern-exclude` | `--inject secret:only=…` / `secret:omit=…` | |
| `exec --require-key` | `--inject secret:require=…` | |
| `exec --env-map-sed` | `--inject secret:rename=…` | |
| `attr`/`set --inject` | rejected | use `--bulk-select` on store commands; `--inject` is exec-only |

YAML `gate: sandbox` in inject policy files is rejected; use `bulk_gate: true`.

## C SDK and CLI

- The C project version is `0.4.0`.
- `exec` builds child environments through layered `--inject LAYER:KIND=SELECTOR`
  rules and optional `--inject-file` YAML policies.
- `exec --bulk-gate` applies the `bulk_select` attribute pre-filter before
  secret supply; gateless exec uses inject pentads only.
- `exec --dry-run` validates inject plans without launching a child; `--json`
  emits machine-readable preflight output, and `--json-summary` reports plan
  plus exit metadata after real executions.
- `export`, `ls`, and `list` use `--bulk-gate` for bulk-gated selection paths.
- Bash completion asks `secdat __completion --bash` for the live command
  surface, including `exec` delegation to underlying command completion after
  option parsing ends.
- ADRs 0002–0005 document the vocabulary and help/prose decisions adopted in
  this release.

## Bindings

- Python, Rust, and Node binding package versions track `0.4.0`.
- The Rust binding requires `libsecdat >= 0.4.0` through `pkg-config`.
- SDK surfaces expose `bulk_select` metadata using the canonical field names.

## Migration checklist

1. Replace `--sandbox-injectable`, `--inject-bulk-gate`, and `--inject-gate`
   with `--bulk-gate` where bulk attribute filtering is intended.
2. Replace `attr`/`set --sandbox-inject` / `--inject-bulk` with
   `--bulk-select`.
3. Rewrite `exec` legacy pattern/require-key/env-map flags as `--inject`
   pentad rules.
4. Update scripts parsing `attr`, `ls --metadata`, or JSON output to read
   `bulk_select` instead of `sandbox_inject` / `inject_bulk`.
5. Reload bash completion after install so `exec` child-command completion is
   available.

## Known constraints

- `exec` inject completion delegates to bash-completion when installed; without
  it, only child command names are completed on the CMD word.
- `named` bulk-select policy remains reserved for a future explicit-key bulk
  flow and is excluded by the current `--bulk-gate` pre-filter.
