# secdat v0.4.2 Release Notes

## Highlights

- Hardens `exec --inject-file` policy parsing so long selector lists are parsed
  dynamically instead of being truncated by fixed-size buffers.
- Documents the supported `--inject-file` YAML subset and its fail-closed
  behavior in CLI help and command references.
- Makes command-line `--inject route:...` rules override policy-file route
  globs, matching the explicit CLI precedence users expect.
- Adds `exec --command-resolution caller-path|child-path|direct` to make the
  executable lookup boundary explicit.

## Security and compatibility

- Oversized or unsupported inject policy syntax now fails closed instead of
  silently dropping selectors.
- The default command resolution mode remains `caller-path`, preserving the
  previous `execvpe`-style lookup behavior for existing scripts.
- `child-path` resolves unqualified command names against the final child
  environment `PATH`, while `direct` requires a slash-qualified executable path.

## Validation

- The release was validated with the full `make check` regression suite.

## Versions

- The C project version is `0.4.2`.
- Python, Rust, and Node binding package versions track `0.4.2`.
- The Rust binding requires `libsecdat >= 0.4.2` through `pkg-config`.
