# secdat v0.2.2 Release Notes

## Highlights

- Improves error recovery by suggesting nearby command names, subcommands, and visible keys when input is mistyped.
- Fixes `unlock --duration ... --until ...` validation so the mutual-exclusion error wins even when either value is malformed.
- Rejects invalid environment variable names before `export` emits shell text or `exec` loads values into a child process environment.
- Adds regression coverage that keeps command-local help usage columns aligned across short, long, meta, and grouped commands.

## C SDK and CLI

- The C project version is `0.2.2`.
- `export` and `exec` now fail before value loading when legacy or mapped key names are not valid environment variable identifiers.
- `--env-map-sed` results are validated as environment variable names, including empty or otherwise invalid mapped names.
- Help and error text remain source-compatible with the `0.2.1` CLI surface, with clearer typo suggestions.

## Bindings

- Python, Rust, and Node binding package versions now track `0.2.2`.
- The Rust binding requires `libsecdat >= 0.2.2` through `pkg-config`.

## Known Constraints

- `export` is intentionally bash-oriented and requires keys to already be valid environment variable identifiers.
- Authentication failures for corrupted or mismatched encrypted entries still require domain/store data recovery rather than automatic repair.
